# 001_sft.md

## 1. 目标

阶段 1 的目标不是把 RL 用上，而是先训练出一个 **可用的 Persona SFT baseline**，让模型具备以下能力：

1. 在单轮上下文下，回复风格明显接近曲曲
2. 在 3~10 轮对话中，保持角色稳定，不快速漂移
3. 回答不仅“像”，还要**答到点上**
4. 不变成只会模仿口头禅的“壳”

本阶段产出应是一个可以进入下一阶段（Preference / DPO）的稳定 SFT 模型。

---

## 2. 训练要求

### 2.1 训练目标

本阶段只做 **监督微调（SFT）**，不做 RL。

训练时优化 4 个子目标：

1. **局部回复像**：当前上下文下，这一轮回复像曲曲
2. **多轮一致**：连续多轮后，语气、结构、立场不漂
3. **价值观稳定**：遇到诱导、模糊、错误前提时，仍保持曲曲的判断方式
4. **自然度**：不是模板机，不像硬 cosplay

### 2.2 输入输出要求

#### 输入
- `system anchor`：简短稳定的人设行为规则
- `history`：到当前轮为止的多轮上下文

#### 输出
- 仅预测 `persona/host` 的真实回复

### 2.3 loss 要求

- **只对 persona reply token 算 loss**
- 不对 user 文本算 loss
- 不对元数据字段算 loss

### 2.4 样本混合要求

推荐混合比例：

- `turn_sft_v1.jsonl`：**60%**
- `conversation_v1.jsonl`：**25%**
- hard / 边角 case（从 turn_sft 中筛高难样本）：**15%**

### 2.5 训练策略要求

采用 curriculum：

- **阶段 A**：短历史（1~2 轮）
- **阶段 B**：中历史（3~5 轮）
- **阶段 C**：长历史（5~10 轮）

目标：先学局部像，再学跨轮稳定。

---

## 3. 数据要求

### 3.1 必需数据

阶段 1 只依赖两类训练数据：

1. `data/05_annotations/conversation_v1.jsonl`
2. `data/05_annotations/turn_sft_v1.jsonl`

对应 schema：

- `docs/annotation_v1/schemas/conversation_record.schema.json`
- `docs/annotation_v1/schemas/turn_sft_record.schema.json`

### 3.2 数据构成要求

#### conversation_v1
用于学习：
- 长上下文承接
- 多轮角色稳定
- 对话节奏

要求：
- 一条样本必须来自一个完整 `call section`
- turn 已按 speaker 合并
- `host -> persona`，`guest -> user`
- topic / split / quality 已标好

#### turn_sft_v1
用于学习：
- 当前上下文下的目标回复方式

要求：
- `history` 必须以 user 结尾
- `target_reply` 必须是 host 的真实回复
- 必须带：
  - `reply_type`
  - `topic_primary`
  - `difficulty`
  - `user_emotion`
  - `transcript_quality`

### 3.3 数据质量门槛

训练集进入 SFT 前必须满足：

1. schema 校验通过
2. 规则 QC 无 fail
3. `transcript_quality` 以 A/B 为主，C 仅少量保留
4. topic 分布不能严重失衡
5. reply_type 不能只集中在 advice / direct_answer

### 3.4 数据切分要求

固定 4 个 split：

- `train`
- `dev`
- `test`
- `holdout`

要求：
- 同一 conversation 不可跨 split
- 同一 episode 尽量不要同时出现在 train 和 holdout
- holdout 专门保留给泛化评估

## 4. 如何量化评测

阶段 1 必须同时做 **训练内指标** 和 **行为指标**。

### 4.1 训练内指标

1. **train loss**
2. **dev loss**
3. **perplexity / token-level NLL**

用途：
- 看模型是否收敛
- 看是否过拟合

但注意：这类指标只能看“会不会拟合”，不能直接代表“像不像”。

### 4.2 行为评测指标

必须建立 SFT 阶段的离线评测集，重点看 5 个指标：

1. **单轮像不像**
   - 在 `turn_sft` test 集上人工或模型评审
   - 看回复风格、判断方式是否像曲曲

2. **多轮一致性**
   - 在 `conversation` test / holdout 上做 5~10 轮续写评测
   - 看第 1 轮到第 5 轮是否漂移

3. **价值观稳定性**
   - 看用户施压、诱导时，是否还保持曲曲立场

4. **用户问题命中度**
   - 不能只像，要答到问题核心

5. **OOC rate（出戏率）**
   - 明显说出曲曲不会说的话的比例

### 4.3 推荐量化口径

建议每轮实验都产出以下分数（0~1 或百分制均可）：

- `style_similarity`
- `structure_similarity`
- `value_alignment`
- `multi_turn_consistency`
- `user_intent_hit`
- `ooc_rate`
- `templatic_rate`

### 4.4 阶段 1 的最低验收门槛

进入阶段 2 前，至少满足：

1. dev loss 已稳定收敛
2. 单轮像不像达到可用水平
3. 多轮 5 轮内不明显漂移
4. OOC rate 低于当前 baseline
5. 模板化率不上升

## 5. 如何确保达到训练目标

### 5.1 不只看 loss

阶段 1 最大风险是：
- loss 很好看
- 但模型只学到表层口头禅

所以必须把 **loss + benchmark + 人工审查** 绑在一起。

### 5.2 样本均衡

要按以下维度做采样均衡：

- `topic_primary`
- `reply_type`
- `difficulty`
- `user_emotion`
- 对话长度

否则模型会学偏：
- 什么都往高频话题上答
- 什么都变成一种固定结构

### 5.3 补 hard cases

阶段 1 就要显式补这些难样本：

1. 用户问题前提错
2. 用户情绪强
3. 用户连续追问 3~4 轮
4. 用户问得很蠢 / 很混乱
5. 用户诱导模型说“很温柔、很模板化”的话

### 5.4 明确 system anchor

训练时建议统一一版稳定 anchor，例如：

- 遇到模糊问题优先澄清或重构
- 优先给框架，再给判断
- 不做廉价鼓励
- 保持现实主义、结果导向
- 不说与长期立场冲突的话

注意：
- anchor 只是帮助稳定训练
- 不能靠夸张 prompt 硬控人设

### 5.5 人工抽检

每轮实验至少抽检：

- 单轮样本：30~50 条
- 多轮样本：20~30 段
- 高压样本：20 条

抽检重点：
- 是否像曲曲
- 是否答到点上
- 是否开始模板化
- 是否越来越“假像”

## 6. 训练实验如何设计

## 6.1 baseline 设计

至少做 3 组实验：

### Exp-1：turn-only baseline
- 只用 `turn_sft_v1.jsonl`
- 目标：验证局部模仿能力

### Exp-2：turn + conversation 混合
- 用推荐混合比例 60/25/15
- 目标：验证多轮稳定性是否提升

### Exp-3：curriculum 版本
- 短历史 → 中历史 → 长历史
- 目标：验证是否降低长对话漂移

## 6.2 固定变量

为了保证实验可比，以下变量尽量固定：

- 基座模型
- tokenizer
- 最大上下文长度
- system anchor
- dev/test/holdout 切分
- 评测脚本

## 6.3 关键对比维度

每轮实验至少比较：

1. 是否加入 `conversation` 样本
2. 是否做 curriculum
3. 是否做 topic / reply_type 均衡采样
4. 是否加入 hard cases 重采样

## 6.4 实验记录模板

每个实验都要记录：

- 实验编号
- 数据版本
- 样本数
- 混合比例
- 训练轮数 / step
- train/dev loss
- style_similarity
- value_alignment
- multi_turn_consistency
- user_intent_hit
- ooc_rate
- templatic_rate
- 主观问题总结

## 6.5 推荐实验顺序

### 第 1 轮
- 先跑 `turn-only baseline`
- 建立最小可用结果

### 第 2 轮
- 加 `conversation`
- 看多轮一致性是否提升

### 第 3 轮
- 加 curriculum + hard case 重采样
- 看长对话和边角 case 是否更稳

---

## 7. 阶段 1 的交付物

阶段 1 完成时，至少交付：

1. 一版 SFT 模型（SFT v1）
2. 固定的 train/dev/test/holdout 切分
3. 一份训练日志与实验表
4. 一套阶段 1 benchmark 评测结果
5. 一份主观错误分析报告

---

## 8. 阶段 1 完成标准

满足以下条件，才进入阶段 2：

1. `turn_sft` 上单轮回复明显像曲曲
2. `conversation` 上 5~10 轮稳定性达到可用水平
3. 用户问题命中度不低
4. OOC rate 可控
5. 模板化率未明显上升
6. 已明确当前最常见失败模式（例如：太软、太模板、价值观漂移、长对话失真）

如果这些条件还没满足，优先继续打磨 SFT 数据和采样策略，不进入 DPO。
